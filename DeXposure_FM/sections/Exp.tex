\section{Empirical Evaluations}
\label{sec:evaluation}

We evaluate DeXposure-FM on three complementary axes: (i) \textit{multi-step forecasting} of edge-level exposures at horizons $h \in \{1, 3, 7\}$ weeks, (ii) \textit{shock analysis} examining model behavior during major stress events (Terra/Luna collapse, FTX failure), and (iii) \textit{imputation} of missing exposures under various masking regimes.

\subsection{Data and Temporal Setup}\label{subsec:data}

We use weekly graph snapshots from March 2020 to July 2025 (approximately 280 weeks). Each snapshot $\mathcal{G}_t = (\mathcal{V}_t, \mathcal{E}_t, \mathbf{X}_t)$ contains:
\begin{itemize}
  \item $|\mathcal{V}_t| \approx 5{,}700$ protocols (mean), with network size evolving over time
  \item $|\mathcal{E}_t| \approx 30{,}400$ directed exposure edges (mean)
  \item Node features $\mathbf{X}_t$: TVL, sector tags, yield metrics
  \item Edge weights: log-transformed USD exposure values
\end{itemize}

\paragraph{Network characteristics.}
The exposure network exhibits heavy-tailed degree distributions with Gini coefficient $\approx 0.70$ and extreme TVL concentration (Gini $\approx 0.99$). Cross-sector exposure ratio averages 0.85, indicating substantial inter-sector connectivity.

\paragraph{Temporal split.}
We adopt a strict temporal split: training on 2020--2024 data ($\sim$230 weeks) and testing on 2025 hold-out data ($\sim$27 weeks). This ensures no future information leakage and reflects realistic deployment conditions.

\subsection{Task I: Multi-step Forecasting}\label{subsec:forecast}

\paragraph{Targets.}
We evaluate forecasts at horizons $h\in\{1,3,7\}$ weeks for:
\begin{enumerate}
  \item \textbf{Link prediction:} predicting whether edge $(i,j)$ exists at $t+h$
  \item \textbf{Edge weight prediction:} forecasting log-exposure $\log(1 + E_{ij,t+h})$
  \item \textbf{Node-level aggregates:} total out-/in-exposure per protocol
\end{enumerate}

\paragraph{Evaluation metrics.}
For link prediction (binary classification on highly imbalanced data):
\begin{itemize}
  \item \textbf{AUPRC}: Area under precision-recall curve (primary metric, robust to class imbalance)
  \item \textbf{AUROC}: Area under ROC curve
  \item \textbf{Recall@$K$}: Fraction of true edges in top-$K$ predictions
\end{itemize}
For weight prediction (regression):
\begin{itemize}
  \item \textbf{MAE}: Mean absolute error on log-transformed weights
  \item \textbf{Weighted MAE}: Exposure-weighted MAE, prioritizing economically material errors
\end{itemize}

\paragraph{Baseline.}
We compare against ROLAND~\cite{You2022ROLAND}, a representative temporal graph learning method that combines recurrent message passing with snapshot-based training. ROLAND uses the same temporal split and feature set.

\paragraph{Results.}
Table~\ref{tab:forecasting} summarizes multi-step forecasting performance. DeXposure-FM (frozen) achieves strong link prediction (AUPRC $>$ 0.93) with remarkable stability across horizons.

\paragraph{Key findings.}
\begin{enumerate}
  \item \textbf{Horizon stability:} Performance remains remarkably stable from $h=1$ to $h=7$, suggesting the model captures persistent structural patterns rather than short-term fluctuations.
  \item \textbf{Class imbalance handling:} High AUPRC despite extreme class imbalance ($<$1\% positive rate) demonstrates effective representation learning.
  \item \textbf{Weight prediction challenge:} MAE $\approx$ 3.3 on log-scale indicates room for improvement in magnitude prediction, though directional accuracy is strong.
\end{enumerate}

\subsection{Task II: Shock Analysis}\label{subsec:shock}

We evaluate model behavior during two major historical stress events:

\paragraph{Terra/Luna collapse (May 2022).}
The algorithmic stablecoin UST lost its peg, triggering a cascade that wiped out $\sim$\$40B in value. We examine:
\begin{itemize}
  \item Pre-shock prediction accuracy (weeks before collapse)
  \item Model's ability to capture rapid network rewiring
  \item Exposure concentration changes in affected protocols
\end{itemize}

\paragraph{FTX failure (November 2022).}
The collapse of centralized exchange FTX caused widespread contagion. We analyze:
\begin{itemize}
  \item Cross-sector exposure propagation patterns
  \item Recovery dynamics in subsequent weeks
  \item Comparison of predicted vs.\ realized exposure changes
\end{itemize}

\paragraph{Evaluation approach.}
For each shock event, we train on pre-event data and evaluate predictions during and after the shock period. We report the same metrics as Task I, stratified by shock phase (pre, during, post).

\subsection{Task III: Imputation}\label{subsec:imputation}

We evaluate the model's ability to reconstruct missing exposure data under controlled masking regimes.

\paragraph{Masking regimes.}
\begin{enumerate}
  \item \textbf{Random masking (MCAR):} Uniformly mask 10\%, 20\%, 30\% of edges
  \item \textbf{Block masking:} Mask contiguous time windows (1--4 weeks)
  \item \textbf{Node masking:} Mask all edges for selected high-degree nodes
\end{enumerate}

\paragraph{Metrics.}
\begin{itemize}
  \item \textbf{Edge recall:} Fraction of masked edges correctly recovered
  \item \textbf{Edge MAE:} Reconstruction error on masked edge weights
  \item \textbf{Node MAE:} Reconstruction error on node-level aggregates
\end{itemize}

\subsection{Ablations and Analysis}\label{subsec:ablations}

\paragraph{Frozen vs.\ finetuned encoder.}
We compare the two training regimes:
\begin{itemize}
  \item \textbf{Frozen:} Pre-trained GraphPFN encoder fixed; only task heads trained (linear probing)
  \item \textbf{Finetuned:} End-to-end training of encoder and task heads
\end{itemize}
Results (Table~\ref{tab:forecasting}) show that frozen encoder achieves strong performance, with finetuning providing marginal gains at higher computational cost.

\paragraph{Hyperparameter sensitivity.}
We conduct Optuna-based hyperparameter search over learning rate, weight decay, hidden dimension, negative sampling ratio, and loss component weights. Key findings:
\begin{itemize}
  \item Learning rate in $[10^{-4}, 10^{-3}]$ is optimal
  \item Hidden dimension 128--256 balances capacity and efficiency
  \item Negative sampling ratio 5--10 handles class imbalance effectively
\end{itemize}
