This section introduces the \sysname{} pipeline.

\subsection{Overview}

\sysname{} is a four-layer pipeline for forecast-grounded DeFi supervision
(Figure~\ref{fig:system}). \textbf{Layer~1: FM Prediction} uses \fmname{} to
forecast near-future credit-exposure graphs. \textbf{Layer~2: Monitor \&
Scenario} converts the forecast distribution into a typed evidence bundle
containing monitor alerts, uncertainty estimates, and stress-scenario losses.
\textbf{Layer~3: LLM Decision} passes this bundle to an LLM, which emits
ranked supervisory tickets with targets, severities, and rationales.
\textbf{Layer~4: Safety Gate} applies data-health and confidence gates to
determine whether intervention-level actions are admissible. The pipeline runs at weekly
cadence on a single RTX~4090, with well under a dollar of LLM API spend
per weekly decision (a full \benchname{} leaderboard run is
approximately \$10).

\subsection{Layer 1: FM Prediction}
\label{ssec:forecaster}

At each decision epoch $t$, the agent observes a credit-exposure snapshot
$G_t=(V_t,E_t,W_t)$, an inter-protocol token-mediated exposure graph defined
in Section~\ref{sec:preliminaries}, together with node covariates $X_t$. For
each forecast horizon $h\in\mathcal{H}=\{1,4,8,12\}$ weeks, it queries
\fmname{} for a predictive distribution
\begin{equation}
\mathcal{P}_{t,h}
\leftarrow
\operatorname{ForecastFM}(G_t,X_t,h).
\end{equation}
From this distribution, the agent constructs an expected-weight predicted
graph $\widehat{G}_{t+h}$ by thresholding edge-existence probabilities at
$\pi_{\min}=0.2$. It also draws $M=50$ Monte Carlo samples to quantify
predictive uncertainty. The predicted-graph construction is defined in
Section~\ref{sec:preliminaries}; implementation choices are given in
Appendix~\ref{app:fm-notation}.

\subsection{Layer 2: Monitor \& Scenario}
\label{ssec:evidence}

Layer~2 converts $\widehat{G}_{t+h}$ and its Monte Carlo samples into a typed
evidence bundle. Five \emph{monitor functionals} summarize concentration,
fragility, and instability in the predicted graph: top-1 PageRank
\citep{BrinPage1998PageRank}, HHI concentration
\citep{Rhoades1993HHI}, network density, PageRank Gini, and degree Gini. An
alert fires when a monitor statistic deviates from its $L=42$ week rolling
baseline by more than $z=2$ standard deviations. Each alert includes two
additional fields: (i) an attribution top-$K$ list of contributing edges,
ranked by marginal contribution, and (ii) an uncertainty-aware confidence
score $C_t(a)\in[0,1]$ that decreases with lower data health, wider Monte
Carlo dispersion, and longer forecast horizons.

In parallel, a \emph{scenario engine} applies five standardized stress
shocks to both $\widehat{G}_{t+h}$ and each Monte Carlo sample: single
protocol failure, bridge cluster failure, stablecoin de-peg, sector lending
shock, and correlated top-$10$ stress. For each scenario $s$ and horizon
$h$, the engine aggregates the worst-half sampled losses as
$\operatorname{CVaR}_{0.5}(s,h)$. Full monitor and scenario definitions are
provided in Appendix~\ref{app:pipeline-math}.

\subsection{Layer 3: LLM Decision}
\label{ssec:llm-decision}

Layer~3 receives the evidence bundle as a typed JSON payload containing
alerts with attribution top-$K$, a scenario-CVaR table, Monte Carlo
dispersion summaries, and the scalar data-health score $\operatorname{DH}_t$.
The LLM is prompted to emit a ranked list of supervisory tickets. Each ticket
contains (a) a severity level from a fixed four-action playbook
(\texttt{Monitor}, \texttt{Investigate}, \texttt{Recommend-Reduce}, and
\texttt{Contingency}), (b) a target set of protocols, and (c) a free-form
rationale that cites specific evidence fields by name. To measure
self-consistency, we repeat the decision call three times at temperature~$0$
and report the Jaccard overlap of ticket targets and severities.

\subsection{Layer 4: Safety Gate}
\label{ssec:gates}

Layer~4 determines whether intervention-level recommendations are feasible.
The \emph{data-health gate} computes
\begin{equation}
\operatorname{DH}_t
=
\operatorname{mean}
\left(
\operatorname{fresh}_t,
\operatorname{miss}_t,
\operatorname{topo}_t,
\operatorname{disc}_t
\right).
\end{equation}
If $\operatorname{DH}_t<\tau_{\mathrm{data}}=0.7$, the system enters safe
mode and emits only \texttt{Monitor} or \texttt{Investigate} tickets,
regardless of the LLM's recommended severity. The \emph{confidence gate}
blocks intervention-level tickets whenever the mean alert confidence
satisfies $\bar{C}_t<\tau_{\mathrm{conf}}=0.6$, even if the data-health gate
passes. Every emitted ticket carries the full evidence bundle, the LLM
rationale, and the gate states that produced it. The resulting audit log is
the unit of release (Section~\ref{sec:lessons}).
