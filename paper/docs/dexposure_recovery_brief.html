DeXposure Recovery Brief

这份页面把项目重新压缩成一句话：DeXposure 不是单个预测模型，而是一套 forecast-measure-decide-gate 框架。 DeXposure-FM 预测未来暴露图，DeXposure-Bench 衡量六个能力轴，DeXposure-Claw 把结构化证据交给 LLM 或规则引擎产出可审计监管 ticket。

主结论：FM 不在每个静态误差指标上赢 persistence，但它提供趋势、校准和鲁棒性信号；LLM 在结构化证据上能提升召回和解释质量，同时必须显式报告 false-intervention cost。

Agent pipeline 怎么做

每周一个决策 epoch。LLM 不直接读原始交易，而是读 FM 预测图、监控指标、压力场景、置信度和 data-health 组成的 typed evidence bundle。

Weekly graph G_t

输入是周度有向加权暴露图和节点特征：protocol 间通过 token 持仓形成 credit exposure。

out: current exposure snapshot

Forecast-FM

构造 G_hat_{t+h}：保留已有边、按 FM residual 重加权，并加入 existence probability 超阈值的新边。

out: predicted graphs + MC samples

Monitor and stress

五个 monitor 指标做 z-score alert；五类 stress scenario 做 CVaR loss 汇总和 target attribution。

out: alerts, CVaR table, confidence

LLM decision

LLM 只在 typed JSON evidence 上输出 target、severity、action 和 rationale，三次 temperature 0 用于一致性。

out: draft supervisory tickets

Safety gates

DH_t < 0.7 或 mean confidence < 0.6 时进入 safe mode，压制 Recommend-Reduce / Contingency。

out: ticket tau* or SafeMode

DeXposure-Bench 是什么

它不是只评预测 MAE，而是把“是否能监管上可用”拆成六个互不替代的能力轴。共同 ground truth 是 4 周后绝对损失 top 5% stressed protocols。

b1_forecast

Temporal graph prediction

PageRank/HHI/Gini MAE, Spearman, trend consistency across h=1,4,8,12.

b2_warning

Streaming early warning

历史事件窗口 Terra/Luna, FTX, SVB/USDC 的 precision、recall、F1、lead time。

b3_calibration

Predictive uncertainty

0.90 prediction interval coverage, width, ECE, CRPS from MC samples.

b4_stress

Scenario fidelity

S1-S5 stress shocks 的 loss MAE、distress count、propagation depth、overlap@k。

b5_decision

Ticket quality

precision、audit completeness/recall、F1、target stability、judge score、FIR。

b6_robustness

Data-quality sensitivity

在低数据、部分图、feature noise/missing 等五类退化下重复预测指标。

Train: 2020-03 to 2024-06, about 222 weeks
Val: 2024-07 to 2024-12
Test: 2025-01 to 2025-08
Dataset: 4,300 protocols, 24,300 tokens, 43.7M exposure entries, 283 weekly snapshots. Decision axes score n=29 weeks because final four test weeks lack t+4 ground truth.

LLM 与 judge 面板

关键不是“LLM 一定更安全”，而是 LLM 决策模型本身影响很大。Judge 轴也有 model-family noise，所以论文把解释质量读作 directional evidence。

Cross-family judge scores

三种 frontier judge 都把 m7 排最高，但 Gemini judge 噪声更大。

Matched-budget precision@k

截断到每周 top-k 后，LLM 每个 target 的命中率约为 rules 的两倍；FM 和 raw-snapshot LLM 在 matched k 下不可区分。

最后怎么讲

这几条是回到论文或答辩时最不容易说偏的版本。

claim 1

Benchmark 的贡献

DeXposure-Bench 把 DeFi agent 从“预测分数”推进到“监管可用性”：召回、解释、FIR、鲁棒性、压力情境都被同一 ground truth 约束。

six axesabsolute-loss ground truthreference methods
claim 2

Pipeline 的贡献

LLM 被限制在结构化证据上推理，不能自由发挥；所有 high-severity action 必须经过 data-health 和 confidence gate，输出可审计。

typed evidencescenario CVaRsafe mode
claim 3

结果的真实含义

FM 和 LLM 都有用，但不是免费的：FM 支持更大的目标集，LLM 提升 F1/解释，Opus 会过度升级，Sonnet 4.6 在当前实验中给出更好的风险-收益点。
