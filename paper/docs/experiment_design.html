DeXposure-Agent · 实验设计

整体框架 + 实现方法的可视化

论文要回答两个问题：(1) FM 骨干能不能给出准确、有校准、有时效的风险信号； (2) 在 FM 信号之上做出来的决策建议，是不是比 naïve / 纯 LLM 基线更靠谱。 为此我们设计了 2 个任务、6 个 benchmark、8 个 method，并按"FM 预测质量 → LLM 推理 → 端到端决策"分三层评估。

§1

两个任务

论文 Section 4 的双轴划分
Task I · 风险监控

FM 骨干能不能产出可用的风险信号？

把当前网络 Gt 喂给 FM，预测未来 Gt+h，从中提取系统性风险指标（HHI / Gini / PageRank / 密度），以及不确定性。看这些信号本身合不合理。

  • 评估对象：FM 预测的图结构 + 网络指标本身
  • 下属 benchmark：b1_forecast、b2_warning、b3_calibration、b6_robustness
  • 对照基线：persistence（Gt+h = Gt）、EvolveGCN、FM-only
Task II · 决策建议

在 FM 信号之上做的 ticket 推荐质量怎么样？

把 FM 预测 + 风险指标 + stress scenario 一起喂给"决策者"（规则引擎或 LLM），让它生成 top-N 行动 ticket，再评估这些 ticket 在真实未来里是否命中了被冲击的协议。

  • 评估对象：action ticket（target 协议 + severity + rationale）
  • 下属 benchmark：b4_stress、b5_decision
  • 对照基线：persistence+rules、纯 LLM(无 FM)、FM+rules、FM+LLM、FM+LLM+gate

§3

六个 benchmark

每个 benchmark 回答一个具体问题
b1_forecast
Risk Forecasting
"预测的网络指标准不准？"
  • HHI / Gini / Density / PageRank MAE
  • Spearman rank correlation
  • Trend consistency（方向一致率）
  • 横跨 h ∈ {1, 4, 8, 12} 周
Layer 1
b2_warning
Early Warning
"能不能在危机前 N 周发警报？"
  • Lead time（提前几周）
  • Precision/Recall @ budget {5,10,20}
  • Alert stability（翻牌率）
  • F1-warning · 在 Terra/FTX/SVB 三个真实事件上评估
Layer 1
b3_calibration
Uncertainty Calibration
"预测的不确定度可信吗？"
  • ECE（Expected Calibration Error）
  • PI coverage（目标 90%）
  • PI width
  • CRPS
  • 用 50 个 MC sample + split conformal 校准
Layer 1
b4_stress
Stress Test
"压力场景下的传染预测对不对？"
  • Contagion loss MAE
  • Distressed count MAE
  • Propagation depth MAE
  • Target overlap@10
  • 5 个标准场景 S1-S5（单点 / 桥 / 稳定币 / 借贷 / top-10 相关冲击）
Layer 3
b5_decision
Decision Quality
"开出去的 ticket 实际兑现吗？"
  • Ticket precision（命中真被冲击的协议）
  • Target stability（相邻周 Jaccard）
  • Audit completeness（覆盖被冲击协议比例）
  • False intervention rate
  • Suppression rate（safe-mode 触发率）
Layer 3
b6_robustness
Robustness
"低数据 / 噪声 / 缺失下还撑得住吗？"
  • 5 种退化情景：low_data_10%, low_data_25%, partial_graph_30, noisy_features_0.1, missing_features_20%
  • 同 b1_forecast 的指标
  • relative_degradation vs clean baseline
