论证主线

1
问题

DeFi 风险通过 tokenized liabilities 和跨协议 exposure 传播，静态规则和通用 LLM 缺少图时间序列预测骨架。

2
方法

framework = forecast -> measure -> decide -> gate，用 FM 预测未来图，用确定性监测和情景引擎生成证据。

3
评测

benchmark = DeXposure-Bench，六轴评估 full pipeline，而不只评估预测或 LLM 单点能力。

4
结论

Claw 是 recall 和 explanation 的 Pareto option，同时 false intervention cost 必须被 gate 和 benchmark 显式管理。

三项贡献

记忆口诀：一个 framework 负责怎么做，一个 benchmark 负责怎么评，一个 Claw 证明怎么部署和取舍。

Contribution I

Forecast-grounded regulatory decision framework

把 DeFi 风险监督定义为四层 forecast -> measure -> decide -> gate pipeline。

  • 预测层：DeXposure-FM 预测未来 exposure graph。
  • 监测层：五个网络风险指标、rolling z-score、alert attribution。
  • 情景层：五个 stress scenarios，CVaR 聚合 tail risk。
  • 决策层：playbook tickets，data-health/confidence gate 管住 intervention。
Contribution II

DeXposure-Bench

标准化评测 forecast-grounded risk-decision pipelines，而不是只测预测或只测 LLM。

  • 六轴：b1 forecast、b2 warning、b3 calibration、b4 stress、b5 decision、b6 robustness。
  • 数据：2020-03 到 2025-08 weekly exposure graphs，严格 temporal split。
  • ground truth：按 absolute exposure loss 选 top 5% stressed protocols。
  • 八个 reference methods 支撑 layer-wise ablation。
Contribution III

DeXposure-Claw

把 framework 打包成可运行、可审计的 agentic system，并在 benchmark 上量化 operating point。

  • 输出 ticket-level evidence bundles 和 audit logs。
  • FM+LLM+gate 把 ticket F1 从 0.0076 提到 0.0233。
  • 解释质量从 2.03 提到 2.69/5。
  • 同时暴露 FIR 约 0.44 的代价，因此不是“全面替代规则”。

Framework 是什么，作用是什么

framework 是 paper 的“系统定义”：规定一个监管 agent 从当前图到最终 ticket 应该经历哪些层，每层产出什么证据。

Input

当前 exposure graph

观测每周 snapshot G_t 和 node covariates X_t。这一步把 DeFi 状态转换成可计算图对象。

Layer 1

FM Prediction

DeXposure-FM 生成 G_{t+h} 的 predictive distribution，并构造 expected graph 与 MC samples。

Layer 2

Monitor & Scenario

用 N1-N5 监测系统性风险，用 S1-S5 压力情景计算 loss、distressed nodes、CVaR。

Layer 3

Decision Policy

规则或 LLM 把 alerts、attribution、scenario evidence 转成 supervisory tickets 和 rationale。

Layer 4

Safety Gate

用 data health 与 alert confidence 决定是否允许 intervention action，否则只输出 monitor/investigate 或 safe-mode notice。

Benchmark 是什么，作用是什么

benchmark 是 paper 的“证据机器”：它不只问模型准不准，还问预测信号能否变成更好的监督决策，以及代价在哪里。

Framework

回答“系统应该怎么工作”。

  • 输入当前图，输出 ticket 或 safe-mode。
  • 定义 forecast、monitor、scenario、decision、gate 的接口。
  • 主要作用：把风险预测转成可审计决策，而不是黑盒 LLM 直接判断。

Benchmark

回答“这样的系统有没有用，哪里有代价”。

  • 用六轴分别压测不同 framework layer。
  • 用 absolute-loss ground truth 贴近监管关注的经济规模。
  • 主要作用：显示 FM 帮在哪里、persistence 强在哪里、LLM 和 gate 的 trade-off 在哪里。

复习提醒：DeXposure-Bench 的 ground truth 是 top_pi=0.05 的 absolute exposure loss stressed set，平均每周约 283 个 protocols。因此 b5 的 recall 和 F1 数值很小，不是系统完全没用，而是 ticket budget 每周只有约 0.7 到 2 个，recall 被预算天然压住。
