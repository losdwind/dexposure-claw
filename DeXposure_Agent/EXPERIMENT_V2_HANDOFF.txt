2. WHAT THE PAPER NEEDS TO PROVE
==================================

Core thesis: "FM + LLM Agent > Pure LLM" and "FM + LLM Agent > FM + Rules"

Four methods to compare:

  C2:     Persistence + Rules      (naive baseline)
  C0:     FM + Rules               (current, has benchmark data)
  C3:     No FM + LLM              (pure LLM, no grounded predictions)
  C0-LLM: FM + LLM                (our full system, the paper's claim)

Key comparisons:
  C0-LLM vs C0   -> proves LLM reasoning > rule engine
  C0-LLM vs C3   -> proves FM grounding makes LLM better
  C0-LLM vs C2   -> proves full system > naive


3. EXPERIMENT DESIGN NEEDED
=============================

Problem: current B5 only tests rule-based decision. Need to evaluate LLM decisions.

3a. Automated LLM evaluation for B5 (29 test weeks)

  For each test week (2025-01 to 2025-08):
    1. Get FM prediction via API: call-api.py predict --date {date} --horizon 4
    2. Compute metrics: call-api.py metrics --date {date} --horizon 4
    3. Run scenarios: call-api.py stress --date {date} --horizon 4
    4. Feed all data to LLM as a structured prompt
    5. LLM outputs: target protocols + action level + rationale (JSON)
    6. Compare LLM targets to ground truth (which protocols actually stressed)

  Implementation options:
    a) Python script calling Claude API (anthropic SDK) in batch
       - Deterministic (temperature=0), reproducible
       - ~29 API calls * ~$0.05 each = ~$1.50 total
       - Can run on any machine with API key
    b) Claude Code subagent via Agent SDK
       - More aligned with "plugin" story
       - Harder to batch-evaluate
    c) Hybrid: Python batch script for data collection, API for decisions
       - Most practical for paper

  For C3 (pure LLM, no FM):
    - Same prompt structure but replace FM predictions with raw text description
    - "Here is the current DeFi network with N protocols, top-5 by TVL are..."
    - No edge probabilities, no predicted graph, no metrics from predicted graph
    - LLM must reason from text description alone

3b. New evaluation metrics for LLM decisions

  Quantitative (same as B5):
    - Ticket precision: fraction of targets that were truly stressed
    - Audit completeness: fraction of truly stressed that were targeted
    - Target stability: Jaccard between consecutive weeks

  New metrics for LLM:
    - Rationale quality: does the rationale cite real numbers from the data?
    - Grounding score: how many claims are traceable to FM output vs hallucinated?
    - Consistency: given same data, does the LLM produce same decision? (run 3x)

  Evaluation method for rationale:
    - LLM-as-judge (GPT-4 or Claude evaluates rationale quality 1-5)
    - Or rule-based: count specific metric citations in rationale text
