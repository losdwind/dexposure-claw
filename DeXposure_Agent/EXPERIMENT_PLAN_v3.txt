PHASE 0: B3 不确定性校准修复 [最高优先级]
=====================================================================

问题诊断:
- 当前 ECE=0.756 (应接近0), PI覆盖率=0.144 (目标0.90)
- 根因: B3的MC采样 (_generate_mc_samples) 使用 N(0, sigma*w) 加噪
  其中 sigma 由 _calibrate_mc_sigma 从训练数据计算
- sigma 的计算方法: std(相邻周边权重相对变化) * sqrt(horizon)
- 但问题在于: sigma 只衡量了边权重的变化,
  没有衡量 *指标层面* 的变化 (M1,M3,M4,M6,M7)
- 边权重的小变化经过 PageRank/HHI/Gini 非线性映射后被放大,
  导致指标层面的PI过窄

修复方案 (两步):

步骤 A: 指标级 sigma 校准
  修改 b3_uncertainty_calibration.py:
  - 不再只校准边权重sigma, 而是直接校准指标层面的变化幅度
  - 新方法: 对训练数据中每周的5个指标计算周间变化的std
  - 用这个指标级std来缩放MC噪声sigma

  具体实现:
  1. 在 _calibrate_mc_sigma 之后, 新增 _calibrate_metric_sigma():
     - 遍历训练期最近52周
     - 每周计算5个指标 M1,M3,M4,M6,M7
     - 计算指标周间变化的std
     - 如果 metric_std > edge_std * 3, 则放大 sigma 使PI更宽
  2. 或者更直接: 用 "conformalized quantile" 方法
     - 在验证集上计算残差分位数
     - 用验证集残差来校正PI宽度

  推荐方案: Conformal Prediction (最简单可靠)
  - 在验证集上跑MC采样 -> 计算PI -> 记录残差
  - 用残差的 (1-alpha) 分位数来扩展PI
  - 保证覆盖率渐近正确

步骤 B: 修改代码
  文件: DeXposure_Agent/experiments/b3_uncertainty_calibration.py
  修改:
  1. 新增 _conformal_calibrate() 函数
  2. 在 run_b3() 中先在验证集上计算 conformal score
  3. 用 conformal score 扩展测试集的PI

预计效果: PI覆盖率应接近 0.90, ECE应降至 <0.10

运行位置: GPU server (需要FM预测)
预计时间: ~10 分钟 (B3本身只需跑1个horizon)

=====================================================================
PHASE 1: EvolveGCN 基线 (C7) [高优先级]
=====================================================================

为什么选 EvolveGCN:
- 最知名的时序图神经网络之一 (Pareja et al., KDD 2020)
- 代码已开源 (torch_geometric_temporal 有实现)
- 直接处理动态图序列,和我们的setting最匹配
- 桩代码已经在 baselines.py 中准备好了

实现方案:

步骤 A: 安装依赖 (GPU server)
  pip install torch-geometric-temporal  # 依赖PyG, 已装

步骤 B: 实现 C7 预测器
  文件: DeXposure_Agent/experiments/competitors/evolvegcn.py (新建)

  架构:
  - EvolveGCN-O variant (ODE-based, 更稳定)
  - 输入: 最近 W=26 周的图序列
  - 输出: 下一周的边权重预测
  - 训练: 在 train+val 数据上端到端训练
  - 推理: 输入测试期图序列, 输出预测图

  训练策略:
  - 滑动窗口 W=26 周作为输入序列
  - 预测 h={1,4,8,12} 周后的边权重
  - 每个horizon训练一个模型 (或一个模型多head)
  - 训练100 epochs, 验证集早停
  - 保存checkpoint到 checkpoints/evolvegcn/

步骤 C: 集成到 predict_helper.py
  - 新增 C7 路由: 加载EvolveGCN checkpoint -> predict
  - 返回 GraphSnapshot 格式 (与C0/C2一致)

步骤 D: 运行 B1, B4, B6
  - B1: 风险指标预测 (MAE + Spearman) 所有horizon
  - B4: 压力测试准确度
  - B6: 鲁棒性 (5个退化场景)
  - 注意: C7 不产生决策, 所以不跑 B2, B3, B5

运行位置: GPU server
预计时间: 训练 ~30-60 分钟, 评估 ~15 分钟

  5. 计算相对于C0基线的性能下降

步骤 B: 在 config.py 中新增字段

  - unconstrained_actions: bool = False  # A8 需要 (暂不跑)

步骤 C: 在 agent_loop.py 中添加 skip_scenario 分支

步骤 D: 新建运行脚本
