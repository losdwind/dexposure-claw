Topic: DeXposure-Agent Plugin Architecture & Code Consolidation

Key Decisions Made
------------------

1. PLUGIN vs EXPERIMENT SEPARATION
   - The Claude Code plugin is the PRODUCT (for end users: DeFi risk managers, researchers)
   - The experiments/ folder is for the PAPER (benchmarks B1-B6, ablations A1-A8, competitor training)
   - They share the same core library: dexposure_agent/
   - An /evaluate skill in the plugin provides lightweight self-evaluation (product feature),
     but the full benchmark suite stays in experiments/ (paper reproducibility)

2. REST API (FastAPI) CHOSEN OVER SSH SCRIPTS
   - Models stay loaded in GPU VRAM — no cold-start per request
   - MC sampling becomes practical (warm model = near-instant)
   - Multiple skills can query concurrently
   - Experiment scripts use the same API endpoints — same numbers guaranteed
   - API surface:
       POST /forecast      -> P_{t,h}
       POST /build-graph   -> G_hat_{t+h}
       POST /mc-sample     -> {G_tilde^m}
       POST /data-health   -> raw freshness/missingness stats
       GET  /models        -> list loaded backbones
       GET  /health        -> server status

3. PYTHON OVER TYPESCRIPT
   - Core agent is PyTorch + NumPy + NetworkX — rewriting in TS would be pointless
   - Claude Code plugins are language-agnostic (markdown + shell + scripts)
   - Plugin layer calls Python REST API via curl / call-api.py

4. CODE CONSOLIDATED INTO DeXposure_Agent/
   - Moved lib/agent/ -> DeXposure_Agent/dexposure_agent/ (standalone Python package)
   - Moved experiments/, plugin/, scripts/, tests/ -> DeXposure_Agent/
   - All imports changed from lib.agent.xxx to dexposure_agent.xxx
   - 71 tests pass with new import paths

5. ARCHITECTURE CONFIRMED: Option B (Hybrid)
   - GPU Server (backend): model inference, MC sampling, raw data health stats
   - Plugin skills (frontend): analytical reasoning — alerting, attribution,
     stress analysis, ticket generation, decision playbook
   - The "agent intelligence" lives in the plugin layer, not the GPU server
