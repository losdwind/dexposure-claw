\section{Experiments}

We evaluate \textsc{DeXposure-Agent} along two axes: (i)~does the forecast-measure pipeline produce accurate, well-calibrated, and timely risk signals (Task~I), and (ii)~do the resulting decision tickets improve upon baselines in precision, stability, and auditability (Task~II)?

\subsection{Task I: Network Risk Monitoring}

\subsubsection{Implementation Details}
\label{sssec:impl_task1}

\paragraph{Data and temporal granularity.}
We use the \textsc{DeXposure} dataset~\cite{wu2025dexposuredatasetbenchmarksinterprotocol}, comprising weekly directed weighted exposure-graph snapshots sourced from the DeFiLlama API. The dataset spans 2020-03 to 2025-08 (283~weeks), with 50--500 protocol nodes and 100--2{,}000 exposure edges per snapshot, totalling over 43.7M data entries. Node features include log-TVL, number of token types, maximum token share, Shannon entropy of token distribution, and protocol category (one-hot, 15~classes). Edge weights are log-scaled USD exposure values. We evaluate multi-horizon forecasting on $\mathcal{H}=\{1,4,8,12\}$ weeks.

\paragraph{Forecast interface and predicted-graph construction.}
For each epoch $t$ and horizon $h\in\mathcal{H}$, we query the backbone model to obtain a predictive distribution $\mathcal{P}_{t,h}$. We construct an expected-weight predicted graph $\widehat{G}_{t+h}$ by thresholding edge existence probabilities at $\pi_{\min}$ and setting $\widehat{w}_{pq} := \Pr(e_{pq}=1)\cdot \mathbb{E}[w_{pq}\mid e_{pq}=1]$ (Eq.~\eqref{eq:expected_graph}). We set $\pi_{\min}=0.2$ on the validation split. For uncertainty quantification, we draw $M=50$ Monte Carlo samples via Bernoulli edge sampling and Gaussian weight perturbation.

\paragraph{Monitoring metrics and alert triggers.}
At each horizon, we compute five monitoring functionals $\mathbf{M}_{t,h}=\Phi(\widehat{G}_{t+h},X_t)$ (Table~\ref{tab:metrics}): systemic importance (M1), HHI concentration (M3), network density (M4), PageRank Gini (M6), and degree Gini (M7). Baseline bands $(\mu_t^{(j)}, \sigma_t^{(j)})$ are computed via a rolling window of $L=26$ weeks. An alert fires when a metric exceeds $z=2$ standard deviations from baseline (Eq.~\eqref{eq:alert_trigger}). Alert confidence follows Eq.~\eqref{eq:alert_confidence}.

\paragraph{Stress scenarios.}
We evaluate five standardised scenarios $\mathcal{S}_0$ (Table~\ref{tab:scenarios}): single protocol failure (S1, 100\% shock on top node), bridge cluster failure (S2, 100\% on bridges), stablecoin de-peg (S3, 50\% on stablecoins), sector-wide lending shock (S4, 30\%), and correlated stress (S5, 20\% on top-10 nodes). CVaR is computed as the mean of the worst $\lambda=0.5$ fraction of MC sample losses (Eq.~\eqref{eq:cvar}).

\paragraph{Attribution.}
For each triggered alert, we rank contributing edges by marginal contribution: $\mathrm{Contrib}(e) := \Phi^{(j)}(\widehat{G}_{t+h}) - \Phi^{(j)}(\widehat{G}_{t+h}\setminus e)$, retaining the top $K=10$.

\paragraph{Evaluation protocol.}
Figure~\ref{fig:eval_framework} summarises the evaluation framework. We evaluate Task~I along three axes:
\begin{enumerate}[leftmargin=*,itemsep=1pt]
    \item \emph{Risk-metric forecasting (B1):} predicted vs.\ realised metrics using MAE and Spearman rank correlation, for each $h\in\{1,4,8,12\}$.
    \item \emph{Early-warning quality (B2):} lead time to stress events, precision/recall under alert budgets $\{5,10,20\}$, alert stability (flip rate), and F1-warning.
    \item \emph{Uncertainty calibration (B3):} expected calibration error (ECE), prediction interval coverage (target 90\%), PI width, and CRPS. We apply split conformal prediction on the validation set to ensure asymptotically valid coverage.
\end{enumerate}

\paragraph{Reproducibility.}
All hyperparameters ($\pi_{\min}$, $L$, $z$, $K$, $M$, $\tau_{\mathrm{data}}$) are selected on the validation split and fixed for test evaluation. Random seeds are fixed for MC sampling; we report mean $\pm$ standard deviation over $R=3$ independent runs.
All benchmark runs reported here use a single NVIDIA GPU (RTX 4080 SUPER 32GB) with Python 3.12, PyTorch 2.4.0+cu124, PyG 2.7.0, and DGL 2.4.0+cu124. The FM backbone checkpoints used by the serving stack are: \texttt{dexposure-fm-h1.pt} (80MB), \texttt{dexposure-fm-h4.pt} (79MB), and \texttt{dexposure-fm-h8-h12.pt} (79MB), with an additional frozen reference checkpoint \texttt{graphpfn-frozen-all-horizons.pt} (79MB). End-to-end benchmark runtime for the full B1--B6 suite (C0+C2+C7) is approximately 60~minutes. The EvolveGCN-O baseline (C7) is trained from scratch on the training split with early stopping on the validation split; training takes approximately 5~minutes per horizon. Backbone training itself is performed offline prior to this evaluation pipeline; therefore, we report checkpoint artifacts and evaluation runtime, while model-training wall-clock is treated as external pretraining cost.

We observe three key patterns. First, all three methods achieve comparable point-forecast ranking quality at $h=4$ (RankCorr: C0 $0.565$, C7 $0.570$, C2 $0.570$) and HHI error (all near $0.032$), indicating that persistence remains a strong short-horizon baseline on coarse concentration metrics and that EvolveGCN does not substantially improve over it. Second, C0 is the \emph{only} method with non-zero trend consistency across all horizons (TrendCons $0.53$--$0.59$), while both C7 and C2 achieve exactly $0.000$. This demonstrates that the FM backbone captures temporal dynamics that neither a trained temporal GNN nor static carry-forward can recover---directional trend detection is the FM's distinctive contribution to the monitoring pipeline. Third, B2 early-warning aggregates are nearly identical across methods (mean precision $0.756$, mean lead time $4.67$~weeks), suggesting that the current alert logic is dominated by shared z-score triggers rather than predictor choice.

For uncertainty calibration, we apply split conformal prediction on the validation set to correct MC-based prediction intervals. After conformal correction, C0 achieves ECE~$=0.019$ and PI coverage~$=0.919$ (target $0.90$), with CRPS~$=0.015$, confirming that the combined MC sampling and conformal calibration strategy produces well-calibrated uncertainty estimates suitable for downstream gating. In B6 robustness, mean relative degradation is moderate for all methods (C0: $0.165$, C7: $0.148$, C2: $0.148$), indicating some sensitivity to distribution shifts and low-data regimes.

\subsection{Task II: Decision-Making}

\subsubsection{Implementation Details}
\label{sssec:impl_task2}

\paragraph{Decision timeline and inputs.}
Task~II operates at the same decision epochs as Task~I. At each epoch, the decision agent receives: (i)~the current state $(G_t,X_t)$, (ii)~multi-horizon forecasts $\{\widehat{G}_{t+h}\}_{h\in\mathcal{H}}$ and optional MC samples, (iii)~monitoring alerts $\mathcal{A}_t$ with evidence and confidence scores, and (iv)~the scenario summary $\mathcal{R}_t$.

\paragraph{Action space and playbook.}
We use a four-level conservative playbook (Table~\ref{tab:playbook}): \texttt{Monitor} (severity weight 0.25, always feasible), \texttt{Investigate} (0.50, always feasible), \texttt{Recommend-Reduce} (0.75, requires $\mathrm{SAFE\_MODE}=0$ and $\ge 2$ alerts), and \texttt{Contingency} (1.00, requires $\mathrm{SAFE\_MODE}=0$ and $\ge 3$ alerts). Intervention-type actions additionally require mean alert confidence $\bar{C}_t \ge \tau_{\mathrm{conf}}=0.6$.

\paragraph{Ticket scoring.}
Tickets are ranked by $\mathrm{Score}_t(u) = w_u \cdot \bar{C}_t \cdot \mathrm{Imp}_t$ (Eq.~\eqref{eq:ticket_score}), where $w_u$ is the playbook severity weight, $\bar{C}_t$ is mean alert confidence, and $\mathrm{Imp}_t$ is the mean CVaR across all scenarios (defaulting to~1.0 when no scenarios are available). We output the top-$N=10$ tickets.

\paragraph{Target selection.}
For intervention tickets, targets are selected from the union of top-$K$ attributed nodes across triggering alerts and worst-case scenarios, ranked by frequency-weighted vote.

\paragraph{Evaluation protocol.}
We evaluate Task~II using both offline and scenario-based criteria:
\begin{enumerate}[leftmargin=*,itemsep=1pt]
    \item \emph{Offline (B5):} ticket precision (fraction followed by real stress), target stability (Jaccard between adjacent epochs), audit completeness (evidence bundle coverage), suppression rate (safe-mode activation), and false intervention rate.
    \item \emph{Scenario-based (B4):} stress-test contagion prediction accuracy---loss MAE, distressed count MAE, propagation depth MAE, and target overlap@$K$ between predicted and actual networks under scenarios S1--S5.
\end{enumerate}

\paragraph{Reproducibility.}
All decision hyperparameters ($K$, $N$, $\tau_{\mathrm{data}}$, $\tau_{\mathrm{conf}}$, $\lambda$) are tuned on the validation split and fixed for testing. MC components use fixed seeds; we report mean $\pm$ standard deviation over $R=3$ runs.

Main observations are as follows. \subsection{Ablation Study}

To validate the necessity of key design components, we conduct four ablation experiments (Table~\ref{tab:ablation}). Each ablation removes one component from the full C0 pipeline and re-evaluates on B5.

\input{sections/Table_Ablation.tex}

The confidence gate (A2) proves essential: removing it increases the false intervention rate from $0.000$ to $0.400$, confirming that confidence-based gating is the primary mechanism preventing spurious high-severity recommendations. The scenario engine (A3) is structurally necessary: without it, the ticket scoring function (Eq.~\eqref{eq:ticket_score}) cannot compute scenario impact $\mathrm{Imp}_t$, causing the decision pipeline to fail entirely. Data-health gating (A1) and multi-horizon diversity (A6) show no measurable impact in the current test window, likely because data quality remains consistently high and the $h=4$ horizon dominates alert generation during this period. These components are expected to become load-bearing under degraded data conditions or during crisis periods with rapid structural shifts.
