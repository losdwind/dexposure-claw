# Summarize Leaderboard

Use this when the user wants benchmark results explained. Always expand compact
method and benchmark IDs on first mention, such as `M5 (FM+Rules)` and
`B5 (Decision Quality)`.

Focus on:

- Which methods are comparable for each benchmark.
- Which results are expected skips.
- Precision/recall trade-offs for decision methods.
- Reproducibility metadata: commit, dirty state, checkpoint hashes, and logs.
