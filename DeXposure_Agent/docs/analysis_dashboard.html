论文论点

把 FM 预测翻译为监管决策需要架构，
整个领域需要一个基准

本文提出 DeXposure-Framework——一个把 FM 预测的 DeFi 曝光图 转译为监管行动票据的四层查询架构；以及 DeXposure-Bench——首个面向 FM+LLM 决策流水线、采用监管对齐真值的标准化评估套件。

架构与基准互为孪生贡献。前者回答"该如何把图预测变成可执行的监管动作"， 后者回答"这种流水线该如何被衡量、如何被复现、如何被社区跨方法比较"。 在 b5_decision 决策质量基准上，完整 FM + LLM 栈实现 F1 = 0.0241， 较纯持久化基线提升 3.2 倍，且其每一层（FM 信号、LLM 判断、安全门控） 都有独立可测的边际贡献。

两项贡献

系统与基准，两条腿走路

它们不是从属关系，而是互为前提：架构在基准下被衡量，基准因架构而被定义。 任何接受 FM+LLM 范式的领域系统，都可以 fork 这套基准做自我评估。

Contribution I
DeXposure-Framework
A four-layer query architecture

把 FM 预测的 DeFi 曝光图，沿 FM 预测 → 监测 + 情景 → LLM 决策 → 安全门控 四层翻译为 监管行动票据。论点：预测图本身不是决策；架构是把信号转译为可执行干预的必经路径。

每层都对应数据上的独立可测贡献——FM 提供 预测方向信号， LLM 提供 跨证据的判断，门控提供 可辩护的输出。

Contribution II
DeXposure-Bench
A community evaluation suite

首个面向 FM+LLM 决策流水线 的领域评估套件。六个基准覆盖预测质量、校准、情景保真、 决策质量、鲁棒性；ground truth 按 监管实务定义（绝对权重损失 top-5%）。

填补了一项空白：通用 LLM 基准（AgentBench、SWE-bench）不评估领域决策； 时序图基准（TGB、OGB）只评 prediction、不评 decision。 本套件是 第一个 把 prediction、decision、calibration、robustness 串成一个 harness 的。

Data card

DeXposure 数据集由 2020 年 3 月至 2025 年 8 月共 283 周的 DeFi 协议曝光图构成， 每周快照记录协议之间的资金敞口边权重。全集合包含 16,610 个去重协议。

Ground truth 由 监管实务定义：对每一周（t）和预测视窗（h）， 计算所有活跃协议在 t → t+h 上的绝对权重损失， 取前 5% 作为"系统性受压"协议集合。该定义对齐监管者关心的系统性影响， 而非比例变化。

§7 主动披露

三条事先说在前头

每一条主动披露约值半个 reviewer goodwill 等级。 论文的可信度，比单个数字漂亮更值钱。

局限一
绝对 recall < 5%

最高 recall 为 0.0123——覆盖真正受压协议池的 1.2%。 这是 ticket_budget = 10 vs |pool| ≈ 283 的机械上限。 建议在论文同时报 precision@k（监管分流的正确框架）， 并在补充材料加 budget 扩展的 robustness 研究。

局限二
LLM 过度干预

m6 / m7 的 FIR ≈ 0.44：将近一半的高严重度 ticket 命中了 非受压协议。Rule-based method 的 FIR = 0。 建议加一张 precision-FIR Pareto 图，把 trade-off 说清楚。

局限三
严重度排序失效

所有 LLM method 的 severity_correlation ≈ 0。 系统能 flag 哪些协议重要，但无法可靠排序多严重。 把它框为 future work，不作为本文结果之一。

    FM 在 aggregate 指标上等于 persistence，但 trend direction 上独赢（0.53 vs 0）。论证：决策层对方向更敏感，trend_consistency 是关键指标。
